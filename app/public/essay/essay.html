An essay on representation

Your angles are lying to you.

A bug we all wrote. · 14 minutes, with figures 
// average two compass headings, in degrees
const avg = (a, b) => (a + b) / 2;

avg(350, 10); // → ?

What do you expect?

The two headings are twenty degrees apart, straddling north. The honest answer is zero — due north, halfway between them.

The code returns one hundred and eighty. The opposite direction. A ship steering by that average runs aground.

02You’ve seen this before.

Not with angles, maybe. But the shape of this bug is familiar. It shows up everywhere we treat a thing as simpler than it is.

Slicing UTF-8 by byte index.
You take s.slice(0, 10) on a string of emoji and get back a question-mark glyph, or worse, mojibake. The bytes are fine. The string is fine. The operation lied about what an index meant.
Gimbal lock.
You parameterize a 3D rotation as three Euler angles. At a particular orientation, two of the axes line up and you lose a degree of freedom. The object is still free to rotate. Your numbers aren’t.
Off-by-one timezone.
A cron job runs twice on the night the clocks fall back, or skips a run when they spring forward. The wall clock isn’t a number line. You wrote it as one anyway.

One-off bugs you remember. But these aren’t one-offs; they’re the same bug, wearing different costumes. Hold that thought.

03The library, and what it gets wrong.

Here’s a small JavaScript units library a colleague might write — clean, well-tested, used in production. Two functions of interest:

// wrap value into [min, max)
export const cyclic = (v, min, max) => {
  const r = max - min;
  return ((v - min) % r + r) % r + min;
};

// linear interpolation
export const lerp = (t, a, b) => a + t * (b - a);

Both of these are right, in the sense that they do exactly what the code says. They are also wrong, in the sense that nobody wanted what the code says. Drag the inputs and watch.

The functions aren’t buggy in the usual sense. There’s no off-by-one, no missed null check. They’re working perfectly on a representation that lies about the data.

04The seam is not where you think it is.

Watch this. The two headings are still 350° and 10°. We’re going to rotate the place we call zero — slide the seam around the dial — and see what happens to the bug.

Sit with that for a moment. The angles never moved. The compass never moved. We changed only one thing — where on the circle we decided to put the cut — and the average moved with the cut.

Whatever this bug is, it is not in the angles. It’s in our decision to flatten a circle into a line and pretend the seam wasn’t there.

05The lift.

So unflatten it. Take the line you wrote the angles on, and let it bend.

That is the whole idea. We didn’t change the math; we changed the surface the math runs on. In the right surface, averaging two close-but-straddling-the-seam points just does the right thing, because they are close in the surface, not far across a fictional gap.

5bThe line was a shadow.

Why did the line need bending? Because it wasn’t the shape of the angle. It was a projection — a flat slice of something taller. Lift two more dimensions and the angle becomes a helix; the line is what you see when you look at the helix from above and throw the rest away.

06What “doing math upstairs” means.

Lightly, then. The lift has a name on paper: each angle θ becomes a point in two dimensions, (cos θ, sin θ). Now angles are vectors on a unit circle, and the operations we wanted come back, geometrically.

6bThe seam, in three dimensions.

Pin a reference angle. Color the whole disk by how far each angle is from it, scalar-style. The result is a smooth bowl with a cliff: a vertical wall along the radial cut at the antipode of the reference, where the math has to climb 360° in zero distance. Move the reference; the cliff moves with it. The cliff isn’t a place. It’s a choice.

07The general pattern.

Step back. The move we just made has a name, and once you have the name you start seeing it everywhere.

When the operations you want don’t behave in your current representation, lift to a richer one where they do, operate there, project back if you have to.

You’ve probably already met it without knowing.

Homogeneous coordinates.
A 2D point becomes a 3D point with a hidden w coordinate. Translation, which is not linear in 2D, becomes a matrix multiply in 3D. Project back by dividing by w. Every graphics pipeline runs on this trick.
Complex numbers.
A real number is a point on a line. A complex number is a point in a plane, and multiplying by one rotates and scales. Rotations of the line aren’t closed; rotations of the plane are. Same trick, older.
One-hot encoding.
“Cat” and “dog” are not 1 and 2; the average of cat and dog is not half-cat-half-dog. Lift each category to its own axis. Distance, addition, gradients all start meaning what you wanted them to mean.
Codepoints over bytes.
A string is not its bytes. Lift to codepoints (or graphemes, one floor up again) and slicing, length, reverse all start telling the truth. The bytes were never the thing.

One pattern, several costumes. The angles example isn’t exotic; it’s just the smallest one where the lift fits on a page.

09What to do about it.

For the library: a Ring type. It stores the value as a 2-vector internally, exposes the same API the scalar version had, and overrides exactly the operations that needed the lift.

class Ring {
  constructor(deg) {
    const r = deg * Math.PI / 180;
    this.x = Math.cos(r);
    this.y = Math.sin(r);
  }
  static mean(rings) {
    const x = mean(rings.map(r => r.x));
    const y = mean(rings.map(r => r.y));
    return Ring.fromVec(x, y);    // renormalize
  }
  get deg() {
    return (Math.atan2(this.y, this.x) * 180 / Math.PI + 360) % 360;
  }
}

That’s the patch. The patch is not the point.

The point is the move. Somewhere in your codebase right now, a quantity that lives on a manifold has been written as a scalar because the type system made that easy. Most of your code on it is fine. A few places — the ones that ask average, close, between — are quietly wrong, and they’ll stay quietly wrong until inputs cross a seam you forgot was there.

Find the seam. Then bend the line.
